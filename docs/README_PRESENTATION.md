## 💡 關鍵話術

### 當被問到「為什麼...」

#### Q: "為什麼空行 Ctrl+D 會關閉連接？"
**A:** 
> "這是 TCP EOF。客戶端執行 shutdown(SHUT_WR)，關閉了寫方向。對 IRC 這種雙向互動協議，客戶端無法發送命令後，保持連接沒有意義。這是正確的協議處理，符合業界慣例。"

#### Q: "為什麼只能有一個 poll()？"
**A:**
> "這是題目要求，也是最佳實踐。所有文件描述符（listener + 所有客戶端）都在同一個 poll 陣列中監聽，確保公平調度，避免某些連接被忽略。"

#### Q: "如何處理非阻塞 I/O？"
**A:**
> "我們把所有 socket 設為 O_NONBLOCK。當 recv() 或 send() 返回 EAGAIN 時，不是錯誤，而是'暫時沒數據/無法寫入'。我們直接返回，等待下次 poll() 通知。這樣可以同時處理多個連接而不阻塞。"

#### Q: "如何確保沒有記憶體洩漏？"
**A:**
> "每個 Client 是動態分配的指針，在 closeClient() 時會 delete。所有 map 容器在析構時自動清理。可以用 valgrind 驗證。而且我們在 Ctrl+Z 測試中，大量消息累積後恢復，沒有洩漏。"
